## 4. Investigation: Intermittent Failures After Run #002

Between #002 (7/7 green) and this run, `Booking › …payment succeeds` and `Reschedule …` occasionally failed with a **blank SPA** mid-flow and element timeouts. Ruled out:

- **Not data pollution**: transactional tables empty at run start; per-email cleanup works.
- **Not admin auth**: Run #002 backend fixes hold; admin cases stable.
- **Not Stripe.js load**: `route.abort()` on `stripe.com` did not stop failures.
- **Not cold-start alone**: with `retries: 2`, three attempts in one run still showed blank pages.

**Root cause**: Reused Vite dev server on `:3000` degraded after long runtime and heavy HMR, **intermittently rendering a blank SPA**. Pattern: retries in the same run all blank; fresh run all green — unrelated to test logic.

**Fix**: Restart frontend dev server → this run 7/7 first try, 0 retries. Kept `retries` and Stripe blocking as extra safety.

## 5. Recommendations

- For long-term stability, let Playwright own a fresh frontend instance (`reuseExistingServer: false` for frontend, `:3000` free), or restart frontend before each suite.
- Keep Stripe.js blocking: full mock payments should not depend on external scripts.
